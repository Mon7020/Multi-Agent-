# 管理员指南

## 适用对象

这份文档面向 `admin` 和 `super_admin`，用于说明后台 Phase 1 已提供的能力、角色边界和日常操作方式。

## 后台入口

- 开发地址：`http://localhost:5174/admin.html`
- 后台 API 文档：`http://localhost:8001/docs`

## 角色边界

### `super_admin`

- 可访问全部后台模块
- 可修改账号角色
- 可修改系统运行参数

### `admin`

- 可访问总览、记忆管理、知识库管理、系统设置、账号管理
- 可修改知识文件发布与显隐
- 可修改运行参数
- 不能执行 `super_admin` 专属角色调整能力

### `operator`

- 可访问总览、记忆管理、知识库管理
- 知识库页仅可查看，不可保存修改
- 无系统设置和账号管理入口

## 模块说明

### 总览

用途：

- 校验当前后台登录角色
- 确认后台模块已经接入

### 记忆管理

用途：

- 按用户搜索记忆数据
- 查看上下文快照和长期偏好
- 修正偏好
- 清理上下文或全部记忆

注意：

- 清理和偏好写入会记录到 `logs/admin_audit.jsonl`

### 知识库管理

用途：

- 查看所有知识文件
- 控制前台显示 / 隐藏
- 控制草稿 / 已发布
- 设置允许访问的角色列表

推荐操作顺序：

1. 上传或准备文件
2. 在后台确认角色范围
3. 设置为前台可见
4. 切换到已发布
5. 用目标角色账号到前台验证

### 系统设置

用途：

- 查看权限模型
- 查看前台只读策略说明
- 更新运行参数

建议修改流程：

1. 先记录当前参数
2. 只调整一个参数组
3. 保存后验证检索与前台页面
4. 若有异常，按原值回退

### 账号管理

用途：

- 查看用户列表
- 调整角色

注意：

- 当前角色修改接口仅 `super_admin` 可用

## 审计日志

默认文件：

- `logs/admin_audit.jsonl`

当前重点记录：

- 后台总览访问
- 记忆写入与清理
- 知识文件访问范围变更
- 系统运行参数变更

排查建议：

1. 先看接口是否返回 `401` 或 `403`
2. 再检查账号角色和状态
3. 最后查看 `admin_audit.jsonl` 是否已经记录操作结果

## 日常检查清单

每天建议确认：

1. `http://localhost:8000/api/v1/health` 正常
2. `http://localhost:8001/health` 正常
3. 后台知识库显隐设置与前台可见结果一致
4. 最近一次参数修改后没有出现明显回归

## 风险提示

- 不要在前台处理知识库发布与显隐
- 不要把运行参数修改交给普通用户
- 调整知识访问角色时，务必验证目标角色能否在前台看到预期文件
