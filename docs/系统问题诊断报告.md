## 问题一：文档数量异常增多

### 定位过程

**代码位置**: [rag_tool.py:905-965](tools/rag_tool.py#L905-L965) - `load_documents()` 方法

**根本原因分析**:

1. **无去重机制**: `load_documents()` 方法在加载文档时**没有检查重复**
```python
# rag_tool.py:937-953
for doc_path in doc_paths:
    # 直接加载，不检查是否已存在
    documents = self._load_single_document(doc_path)
    all_documents.extend(documents)
```

2. **增量加载时重复添加**: `_add_documents_to_db()` 每次调用都会追加文档
```python
# rag_tool.py:967-987
def _add_documents_to_db(self, documents):
    for doc in documents:
        # 直接 add，不检查 id 是否已存在
        self.collection.add(ids=[doc_id], ...)
```

3. **CRAG定时刷新导致重复**: CRAG 刷新时会重新加载所有文档，但不清除旧数据

### 解决方案

**短期修复** - 在加载前检查并跳过已存在的文档:
```python
def load_documents(self, force_reload=False):
    # 获取现有文档ID集合
    existing_ids = set()
    if self.collection is not None:
        try:
            existing = self.collection.get(include=["metadatas"])
            existing_ids = set(existing["ids"]) if existing else set()
        except:
            pass
    
    for new_doc in documents:
        if new_doc.id not in existing_ids:  # 去重检查
            all_documents.append(new_doc)
```

**长期修复** - 实现基于 content_hash 的去重机制:
```python
import hashlib

def _compute_doc_hash(content: str) -> str:
    return hashlib.md5(content.encode('utf-8')).hexdigest()

# 在metadata中存储hash
metadata = {"file_path": path, "content_hash": hash_val}
```

## 问题二：已删除文档仍能被检索到

### 定位过程

**代码位置**: 
- [rag_tool.py:1045-1077](tools/rag_tool.py#L1045-L1077) - `delete_documents_by_source()`
- [rag_tool.py:1020-1038](tools/rag_tool.py#L1020-L1038) - `_invalidate_cache()`

**根本原因分析**:

1. **BM25索引未同步更新**: 删除文档后只调用了 `_invalidate_cache()` 清除Redis/内存缓存，但 **BM25的内存索引未重建**
```python
# rag_tool.py:1069 - 只清除了缓存
self._invalidate_cache()  
# 但 BM25索引中仍有被删除的文档！
```

2. **Redis缓存TTL过长**: 缓存默认1小时TTL，删除后缓存仍有效
```python
# redis_cache_manager.py
default_ttl = 3600  # 1小时，期间旧结果仍可命中
```

3. **向量检索有源文件校验但BM25没有**:
```python
# 向量检索 - 有文件存在性检查 ✓
if source_file and not os.path.exists(source_file):
    continue  # 跳过已删除文件

# BM25检索 - 无此检查 ✗
bm25_results = self.bm25_retriever.search(query)  # 返回所有匹配
```

### 解决方案

**修复1** - 删除文档时同时重建BM25索引:
```python
def delete_documents_by_source(self, source_file: str) -> int:
    # ... 删除ChromaDB文档 ...
    
    # 新增：清除BM25索引中的对应文档
    self._bm25_indexed = False  # 标记需要重建
    self._bm25_documents = []   # 清空BM25文档列表
    
    self._invalidate_cache()
    return len(ids_to_delete)
```

**修复2** - BM25检索时增加源文件过滤:
```python
# 在BM25搜索结果中过滤已删除文档
filtered_results = []
for r in bm25_results:
    source = r.metadata.get("source_file", "")
    if source and os.path.exists(source):  # 文件存在才保留
        filtered_results.append(r)
```

**修复3** - 降低缓存TTL或使用版本号失效:
```python
# 使用文档版本号作为缓存键的一部分
cache_key = f"{query}:{doc_version}:{hashlib.md5(query.encode()).hexdigest()}"
```

## 问题四：Collection does not exist 错误

### 定位过程

**错误信息**:
```
Collection a52d6214-d4f0-4f0d-909f-6546015c2b67 does not exist
```

**代码位置**: [rag_tool.py:785-810](tools/rag_tool.py#L785-L810), [rag_tool.py:1398-1420](tools/rag_tool.py#L1398-L1420)

**根本原因分析**:

1. **持久化数据目录损坏或丢失**: ChromaDB 数据存储在 `chroma_data/` 目录
```python
# rag_tool.py:777-778
chroma_data_path = os.path.join(project_root, 'chroma_data')
os.makedirs(chroma_data_path, exist_ok=True)
self.chroma_client = chromadb.PersistentClient(path=chroma_data_path)
```

2. **集合ID与名称不匹配**: 错误消息显示的是UUID格式ID (`a52d6214...`)，但代码中使用的是**名称** (`_collection_name`)
```python
# 使用名称获取
self.collection = self.chroma_client.get_collection(self._collection_name)
# 但错误显示的是 UUID，说明某处可能直接使用了 ID
```

3. **并发访问竞争条件**: 多个请求同时操作集合可能导致状态不一致

### 解决方案

**增强版 ensure_collection_ready 方法**:
```python
def ensure_collection_ready(self) -> bool:
    """确保集合可用，如果不存在则重建"""
    if self.chroma_client is None:
        return False
    
    try:
        # 尝试通过名称获取
        self.collection = self.chroma_client.get_collection(self._collection_name)
        return True
    except Exception as e:
        app_logger.warning(f"集合 {self._collection_name} 不存在: {e}")
        
        # 尝试列出所有集合进行诊断
        try:
            collections = self.chroma_client.list_collections()
            app_logger.info(f"当前存在的集合: {[c.name for c in collections]}")
        except Exception as list_err:
            app_logger.warning(f"无法列出集合: {list_err}")
        
        # 重建集合
        try:
            # 先尝试删除残留数据
            try:
                self.chroma_client.delete_collection(self._collection_name)
            except:
                pass
            
            # 创建新集合
            self.collection = self.chroma_client.create_collection(
                name=self._collection_name,
                metadata={
                    "hnsw:space": "cosine",
                    "hnsw:construction_ef": 128,
                    "hnsw:M": 32
                }
            )
            
            # 标记需要重新加载数据
            self._need_initial_load = True
            self._db_available = True
            self._invalidate_cache()
            
            app_logger.info(f"集合已重建: {self._collection_name}")
            return True
        except Exception as create_err:
            app_logger.error(f"重建集合失败: {create_err}")
            self._db_available = False
            return False
```
