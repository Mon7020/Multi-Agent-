# 智能客服Agent测试指南

## 概述

本测试套件用于全面评估智能客服Agent的三大核心能力：
1. **检索能力** - 从知识库中准确查找信息
2. **上下文能力** - 在多轮对话中保持连贯性
3. **沟通能力** - 使用恰当的话术和沟通技巧

## 评估标准

### 1. 关键词匹配度 (40%)
- 检查回答中是否包含期望的关键词
- 计算公式：`匹配关键词数 / 总期望关键词数`

### 2. 意图识别准确率 (30%)
- 检查Agent识别的意图是否与期望一致
- 意图类型：price_inquiry, recommendation, comparison, sales, general, greeting, farewell, negotiation, tech_support, product_spec

### 3. 响应质量 (30%)
- **长度适中**：50-300字（0.3分）
- **结构清晰**：包含标点、分段（0.3分）
- **专业术语**：使用行业术语（0.2分）
- **礼貌用语**：包含请、谢谢等（0.2分）

### 4. 上下文连贯性 (额外加分)
- **指代识别**：正确识别"第二款"、"刚才说的"等指代（0.4分）
- **话题连贯**：前后轮次有共同词汇（0.3分）
- **逻辑连贯**：使用"刚才"、"之前"等连接词（0.3分）

## 评分等级

| 得分范围 | 等级 | 说明 |
|---------|------|------|
| 90-100 | 优秀 | 表现卓越，完全满足要求 |
| 80-89 | 良好 | 表现良好，基本满足要求 |
| 70-79 | 中等 | 表现一般，部分需要改进 |
| 60-69 | 及格 | 勉强满足最低要求 |
| <60 | 不及格 | 需要重大改进 |

## 测试运行

### 环境要求
- Python 3.10+
- 项目依赖：`pip install -r requirements.txt`
- Agent服务运行中（后端API）

### 运行命令

```bash
# 进入项目目录
cd D:\agentlearn\ai-engineer-training\projects\test2langchain

# 运行测试套件
python agent_test_suite.py
```

### 测试模式选择

运行时会提示选择测试模式：
1. **全部测试** - 运行所有20个测试用例
2. **仅测试检索能力** - 运行R001-R005, I001, I004, I005
3. **仅测试上下文能力** - 运行C001-C005, I003
4. **仅测试沟通能力** - 运行M001-M005, I002

### 输出结果

测试完成后会生成：
1. **控制台报告** - 显示详细测试结果
2. **JSON文件** - `test_results_YYYYMMDD_HHMMSS.json` 包含原始数据

## 测试用例设计原理

### 检索能力测试设计
1. **精确检索**：直接查询具体产品价格（R001）
2. **语义检索**：理解"适合游戏"的语义（R002）
3. **对比检索**：同时检索多个产品信息（R003）
4. **政策检索**：从FAQ中查找具体条款（R004）
5. **条件检索**：结合价格约束进行筛选（R005）

### 上下文能力测试设计
1. **指代消解**：测试"第二款"的识别（C001）
2. **产品记忆**：记住前文讨论的产品（C002）
3. **列表记忆**：记住推荐列表的顺序（C003）
4. **逻辑延伸**：基于前文信息推理（C004）
5. **功能记忆**：记住产品的具体功能（C005）

### 沟通能力测试设计
1. **基本礼仪**：问候和结束语（M001, M002）
2. **谈判技巧**：处理价格异议（M003）
3. **需求挖掘**：引导客户表达需求（M004）
4. **疑虑处理**：打消客户顾虑（M005）

## 故障排除

### 常见问题
1. **Agent服务未启动**
   ```
   错误：连接被拒绝
   解决：确保后端服务正在运行（uvicorn app.main:app）
   ```

2. **知识库未加载**
   ```
   错误：检索结果为空
   解决：检查data/目录下的文件是否存在且格式正确
   ```

3. **测试超时**
   ```
   错误：请求超时
   解决：检查网络连接，或增加超时时间设置
   ```

### 调试建议
1. 启用详细日志：设置日志级别为DEBUG
2. 检查RAG检索结果：查看检索到的文档内容
3. 验证意图分类：检查SupervisorAgent的意图识别结果
4. 监控会话上下文：查看SessionContext的状态变化

## 扩展测试用例

如需扩展测试用例，请遵循以下原则：

### 新增检索测试
1. 选择知识库中的具体信息点
2. 设计自然语言查询
3. 确定期望的关键词
4. 标注难度等级

### 新增上下文测试
1. 设计两轮相关的对话
2. 第一轮建立上下文
3. 第二轮测试上下文依赖
4. 明确指代关系

### 新增沟通测试
1. 选择典型的客服场景
2. 设计客户的实际表达
3. 确定期望的回应方式
4. 关注话术和语气
